## 1. 문서 개요

### 1.1 목적

본 문서는 TrackCar Mobile App의 요구사항과 기능을 정의한다. 일반 고객(owner/staff)이 모바일에서 차량 관제, 운행 조회, 조직 운영을 직접 수행할 수 있는 Self-Service형 차량관제 앱의 MVP 범위를 명확히 한다.

### 1.2 설계 기준

- 모바일 앱은 고객용 운영 콘솔 + 관제 앱으로 설계한다.
- Internal Admin Web는 내부 운영자(설치기사/운영자) 전용으로 분리한다.
- Owner는 모바일 앱에서 자기 조직의 담당자, 그룹, 차량-기사 연결을 직접 관리한다.
- Staff는 조회/알림 중심의 단순 권한으로 제한한다.
- 차량-기사 관계는 일자별 배차가 아니라 상시 연결(Assignment Link) 기준으로 관리한다.
- 차별화 기능은 "할당 상태 기반 관제 홈"으로 정의한다.

### 1.3 참고 기준

- 올빗 모바일 앱: 대시보드, 관제현황, 운행관리, 차량알림, 마이올빗 구조
- DTG/eTAS 관련 문서: 운행기록 전송 내역, 성공률, 상태정보 표출 필요성
- Backend 아키텍처 문서: 저장소 계층, 실시간 수집, 배치 전송 구조

## 2. 사용자 유형 및 권한

### 2.1 사용자 유형

| 유형 | 설명 | 역할 |
|------|------|------|
| **Owner** | 조직 대표 또는 운영 책임자 | 조직 운영, 담당자 관리, 그룹/차량/기사 관리 |
| **Staff** | Owner가 생성한 담당자 계정 | 조회/알림 중심, 권한 범위 내 데이터만 접근 |
| **Internal Admin** | 설치기사/운영자/검증담당자 | 내부 Admin Web 전용, 본 문서 범위 제외 |

### 2.2 Owner 권한

- 조직 정보 조회
- 담당자 계정 생성/비활성화
- 그룹 생성/수정/비활성화
- 차량 그룹 지정/변경
- 기사 등록/수정/비활성화
- 차량-기사 연결/해제
- 알림 수신 대상 지정
- 입·출차 위치 설정
- 차량 상태/운행 이력/알림 전체 조회

### 2.3 Staff 권한

- 허용된 그룹/차량 조회
- 차량 상세 조회
- 운행 이력 조회
- 알림 조회 및 수신
- 개인 프로필/알림 설정 변경

### 2.4 Staff 제한사항

- 담당자 계정 생성 불가
- 그룹 생성/삭제 불가
- 차량/기사 master 수정 불가
- 차량-기사 연결 관리 불가
- 시스템 설정 변경 불가

## 3. 핵심 관리 객체

### 3.1 엔터티 목록

| 객체 | 설명 | eTAS 관련 |
|------|------|-----------|
| **Owner** | 조직 대표 계정 | - |

| **Group** | 차량 운영 단위 | - |
| **Vehicle** | 차량 정보 | vehicle_type (자동차유형코드), vin, vehicle_no |
| **Driver** | 기사 정보 | driver_code (운전자코드) |
| **Device (DTG)** | 디지털 운행기록장치 | serial_no, approval_no |
| **Assignment Link** | 차량-기사 상시 연결 관계 | driver_code 연동 |
| **Geofence** | 입·출차 위치 | - |
| **Alert** | 알림 | - |
| **Trip** | 운행 이력 | driver_code, 운행거리/시간 |
| **Transmission Status** | 전송 상태 | eTAS 전송 결과 |

### 3.2 차량 데이터 요구사항

- 차량번호: `[별표 4]` 차량번호 형식 검증
- VIN: 17자리 영문/숫자 (I, O, Q 제외)
- 자동차유형코드: TRUCK, BUS, TAXI, SPECIAL, ETC (eTAS 전송용)
- DTG 연결 상태: 실시간 통신 상태

### 3.3 기사 데이터 요구사항

- 기사명, 연락처
- 운전자코드: 최대 20자리, 영문/숫자 (eTAS 전송용, [별표 2] 참조)
- 연결 차량 정보

## 4. 기능 요구사항

### 4.1 공통 기능

| 기능 | Owner | Staff | 설명 |
|------|-------|-------|------|
| 관제 홈 (대시보드) | O | O | 할당 상태 기반 KPI 카드, 예외 차량, 최근 알림 |
| 차량 목록/상세 | O | O | 검색/필터, 상태 뱃지, 기사 연결 상태 |
| 운행 이력 | O | O | 기간별 조회, 요약, 상세 경로 |
| 알림 센터 | O | O | 심각도/상태 필터, 읽음 처리, 일괄 확인 |
| Workspace | O | X | Owner 전용 운영 관리 공간 |
| 내 정보/설정 | O | O | 프로필, 알림 설정, 앱 정보 |

### 4.2 Workspace (Owner 전용)

| 기능 | 설명 |
|------|------|
| 담당자 관리 | 계정 생성, 목록, 상세, 활성/비활성, 접근 범위 설정 |
| 그룹 관리 | 생성, 수정, 비활성화, 차량 배정 |
| 기사 관리 | 등록, 수정, 비활성화, 연결 차량 조회 |
| 차량-기사 연결 관리 | 연결/해제, 미연결 차량 조회 |
| 입·출차 위치 관리 | Geofence 설정, 알림 수신자 지정 |

### 4.3 관제 홈 기능 상세

**주요 표시 정보:**
- 전체 차량 수
- 실시간 운행 차량 수
- 정차 차량 수
- 오프라인 차량 수
- 경고 차량 수
- 담당자 연결 차량 수
- 담당자 미연결 차량 수
- 최근 주요 알림 요약
- 그룹 필터 적용 가능

**처리 규칙:**
- Owner는 조직 전체 또는 선택 그룹 기준으로 조회
- Staff는 권한 범위 내 차량만 조회
- 미연결/오프라인/경고 차량은 예외 항목으로 강조

### 4.4 차량 관리 기능 상세

**목록 기능:**
- 리스트 형태 조회
- 상태 필터 (REGISTERED, DEVICE_LINKED, DRIVER_LINKED, VERIFIED, ACTIVE, INACTIVE)
- 운행 상태 필터 (RUNNING, IDLE, OFFLINE, WARNING)
- 그룹 필터
- 검색 (차량번호/별칭/기사명)

**상세 기능:**
- 차량 기본정보 (vehicle_no, vin, vehicle_type)
- 그룹 정보
- 현재 담당 기사 (driver_code 포함)
- 장치 연결 상태
- 최근 위치/수신 시각
- 오늘 운행 요약
- 최근 알림
- Owner: 그룹 변경, 기사 연결 변경 가능
- Staff: 조회만 가능

### 4.5 운행 이력 기능 상세

**조회 항목:**
- 운행 시작/종료 시각
- 운행 시간
- 운행 거리
- 평균 속도, 최고 속도
- 정차 시간
- 연결 기사 (driver_code 포함)

**조회 조건:**
- 오늘/어제/최근 7일 빠른 조회
- 차량/그룹/기사별 필터
- 최대 93일 조회 가능

### 4.6 알림 기능 상세

**알림 유형:**
- 오프라인 알림
- 전송 실패 알림
- 이상 상태 알림
- 입차/출차 알림
- 위험운전/경고 알림
- 담당자 미연결 예외 알림

**알림 심각도:**
- CRITICAL: 심각
- HIGH: 높음
- MEDIUM: 보통
- LOW: 낮음

**처리 기능:**
- 개별/일괄 읽음 처리
- 중요 알림 고정
- 차량 상세로 이동

## 6. 비기능 요구사항

### 6.1 보안

- 조직 간 데이터는 논리적으로 완전 분리
- Owner는 자기 조직 데이터만 관리 가능
- Staff는 부여받은 범위만 조회 가능
- 내부 운영 기능은 일반 사용자에게 노출 불가
- 장치 식별값, 인증 정보는 일반 사용자 수정 불가

### 6.2 실시간성

- 실시간 화면: 30초 polling + 수동 새로고침
- 마지막 갱신 시각 표시
- 오프라인 차량은 카드 상단 붉은 상태 표시

### 6.3 확장성

- 향후 일자별 배차 기능으로 확장 가능
- 향후 다중 기사/교대 운전 시나리오 수용
- 향후 eTAS 제출/전송 결과 조회 기능 연계

### 6.4 운영성

- 담당자 생성/비활성화 이력 감사 로그
- 그룹 생성/수정 이력
- 차량-기사 연결 변경 이력
- 알림 정책 변경 이력

## 8. 성공 기준

### 8.1 업무 성공 기준

- Owner가 서비스 도움 없이 담당자와 그룹을 모바일에서 관리할 수 있다.
- Owner가 차량-기사 연결 상태를 스스로 운영할 수 있다.
- 담당자는 조회/알림 중심으로 빠르게 사용할 수 있다.
- 차량 상태, 알림, 운행 이력을 앱에서 즉시 확인할 수 있다.

### 8.2 제품 성공 기준

- 개인 사용자와 회사 사용자를 하나의 앱 구조로 수용한다.
- DTG/eTAS 연계 확장성을 해치지 않는 구조를 가진다.
- Owner와 Staff의 기능 차이는 UI에서 명확히 구분된다.
